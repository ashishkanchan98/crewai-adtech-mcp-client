 Single Agent vs Multi-Agent
  
  Single agent (LangGraph version): One LLM loop calls all tools itself, writes one answer.

  Multi-agent (CrewAI version): Multiple independent LLM instances, each with a specific role, tools, and goal. They hand work to each other.

  Where Multi-Agency Lives in the Code
  
  app/crew/tasks.py — each task is assigned to a different agent:
  campaign_check = Task(agent=agents["campaign"], ...)   # LLM #2
  audience_check = Task(agent=agents["audience_creative"], ...)  # LLM #3
  deal_check     = Task(agent=agents["deal_inventory"], ...)     # LLM #4
  
  synthesis = Task(
      agent=agents["triage"],           # LLM #1 again
      context=[campaign_check, audience_check, deal_check],  # sees ALL outputs
  )   
  
  app/crew/agents.py — each agent has different tools, so they literally cannot do each other's job:
  agents["campaign"]    # can call getCampaignStatus — cannot call getDealStatus
  agents["deal_inventory"]  # can call getDealStatus — cannot call getCampaignStatus
  
  app/crew/crews.py — CrewAI runs each task through its assigned agent's LLM:
  crew = Crew(
      agents=[triage, campaign, audience, deal],  # 4 LLM instances
      tasks=[triage_task, campaign_task, audience_task, deal_task, synthesis_task],
      process=Process.sequential,
  )   
  crew.kickoff()  # each task → different LLM with different tools
